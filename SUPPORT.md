# Support

Use public GitHub issues for reproducible server bugs, focused feature requests,
documentation gaps, and adapter or protocol compatibility reports.

Before opening an issue:

1. Check the README and architecture documentation for the expected source-folder
   adapter and endpoint contract.
2. Try the smallest public reproduction that does not include credentials,
   private endpoint URLs, private paths, raw sensitive wiki content, or
   screenshots exposing private infrastructure.
3. Use the issue form that matches the request.

For suspected vulnerabilities, do not open a public issue. Follow
[SECURITY.md](SECURITY.md).

For compatibility reports, include the source-folder format, command or endpoint
used, and whether the issue affects HTTP, MCP JSON-RPC, A2A, CLI, graph, search,
or read behavior.
